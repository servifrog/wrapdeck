WrapDeck Security summary

Security And Privacy Summary

For plain-English review of the current static WrapDeck app.

Short version: WrapDeck is a static, session-only note assembly page. The app code does not save, upload, analyze, or remember what is typed.

Architecture

  • Static first-party HTML, CSS, and JavaScript.
  • No backend application server.
  • No admin login or admin routes.
  • No accounts, OAuth, API endpoints, or database.
  • No AI service or CRM automation.

Data Handling

  • Templates, Quick pieces, notes, custom order, and scratchpad text live in the current browser tab's temporary memory.
  • Refreshing or closing the tab clears the session.
  • WrapDeck does not use cookies, localStorage, sessionStorage, IndexedDB, Cache API storage, or service worker registration.
  • WrapDeck does not read clipboard contents.
  • Copy actions write text to the clipboard only after the user clicks an explicit Copy/New Call/Clear action.

Network Behavior

  • After the page loads, WrapDeck's app code does not make runtime network requests.
  • The Content Security Policy uses connect-src 'none' to block script-initiated connections.
  • No remote fonts, images, scripts, stylesheets, analytics pixels, or third-party runtime assets are used by the app code.
  • The static host can still receive ordinary request metadata when someone loads a page, like any website host.

Browser Controls

  • Content Security Policy blocks inline scripts, object/media/frame loads, worker loads, form submission, and script connections.
  • Deployment headers include no-store and no-transform so the host should not cache or rewrite the HTML.
  • Permissions Policy disables clipboard-read and other browser capabilities the app does not need.
  • Search-engine directives request no indexing, but they are not access control.

Limits

  • This summary does not claim workplace authorization or suitability for sensitive information.
  • Managed browsers, extensions, devices, networks, screenshot tools, and destination systems may still observe user activity outside WrapDeck's control.
  • Official records should remain in the usual systems and workflows.
