WrapDeck Privacy policy

Privacy Policy

Effective and last updated: June 24, 2026

Plain-language summary: WrapDeck processes templates, Quick Forms, Note Blocks, Scratchpad text, saved non-call reference text, and notes only inside your current browser tab. The app does not upload, save, sync, analyze, sell, or share the text you enter.

What WrapDeck processes

WrapDeck processes the templates, Quick Forms, Note Blocks, saved non-call reference text, custom details, transfer destinations, Scratchpad text, and dock edits that you enter so it can assemble plain text in your browser. This processing happens in the browser tab's temporary memory.

What WrapDeck does not collect

  • No accounts, names, email addresses, or login information.
  • No analytics, tracking pixels, advertising systems, or behavioral profiles.
  • No cookies, local storage, session storage, IndexedDB, saved history, or service worker.
  • No uploaded templates, note contents, Quick Forms, Note Blocks, saved reference text, Scratchpad text, setup strings, or dock edits.
  • No clipboard reading.

Temporary session data

Templates, Quick Forms, Note Blocks, saved reference text, Scratchpad text, current notes, and manual edits exist only in the current browser tab's memory. Refreshing or closing the tab destroys this session data. WrapDeck does not provide recovery or history. When session data is present, WrapDeck asks the browser to warn before a refresh or tab close, but browsers may choose how or whether to display that warning. If you explicitly save a generated Build As You Go note as a template, it becomes reusable Full Template text for this tab session.

Setup strings

Setup strings beginning with WDSTRING_V1 export reusable workbench state, including saved non-call reference text and Full Templates explicitly saved from Build As You Go, as plain text after an explicit user action. They do not save to the browser, disk, backend, or network. WDSTRING_V1 setup strings must not automatically include current-call note content, scratchpad text, transfer destinations, or active Quick Form content.

Clipboard use

WrapDeck writes plain text to the system clipboard only after explicit user actions. Each copy action has a bounded target, such as the current call note, an active Quick Form, a setup string, Full Templates, Note Blocks, saved non-call reference text, or a valid Full Template copied as is. New Call + Clear Clipboard and Clear Entire Session attempt to overwrite the clipboard with an empty value. Your browser or device may block clipboard access. WrapDeck does not read clipboard contents.

Network activity and hosting logs

After the page is loaded, WrapDeck's app code does not make network requests. Its Content Security Policy blocks script-initiated connections. WrapDeck is hosted on Cloudflare Pages. Cloudflare may receive ordinary request metadata when you open this page or the privacy policy, such as an IP address, browser information, date, and time. Cloudflare's privacy and retention practices apply to hosting request metadata. WrapDeck does not add app analytics, tracking pixels, or client-side telemetry.

Security controls

WrapDeck uses a restrictive Content Security Policy and deployment response headers intended to block runtime network connections, inline code, framing, browser caching, referrer leakage, and search indexing. Search-engine directives are not access control, and no browser control can prevent observation by software or people with access to the device.

Managed devices and systems

WrapDeck cannot control or prevent monitoring performed by a managed browser, browser extension, computer, network, security product, screenshot tool, or destination system. Applicable policies and review requirements still apply.

Sensitive information

WrapDeck is for temporary note assembly in the current browser tab. Do not use WrapDeck as a records system, storage system, or source of truth. Keep official records in the usual systems and workflows.

Sharing, selling, and automated decisions

WrapDeck does not sell or share personal information, use note contents for advertising, train an AI model, or make automated decisions about people.

Browser privacy signals

WrapDeck does not use cross-site tracking, targeted advertising, analytics, cookies, or sale/sharing of personal information. Because the app does not perform those activities, browser privacy signals such as Do Not Track or Global Privacy Control do not change the app's behavior.

Changes to this policy

If WrapDeck's data practices change, this policy must be updated before the changed version is deployed. The effective date at the top identifies the current version.

Operator and contact

For privacy or security questions, email privacy [at] wrapdeck.app. For general feedback, email feedback [at] wrapdeck.app.
